Project Overview

NetSim is a modular, extensible, real-time network simulator built with Python 3.x and asyncio. It provides an event-driven simulation engine for modeling packet-level behavior across hosts, routers, and switches — complete with ARP resolution, IP forwarding, ICMP echo/reply, and static routing with longest-prefix match.

A built-in aiohttp WebSocket server streams real-time telemetry to a web dashboard powered by vis-network for interactive topology visualization. Scenarios are defined in JSON or YAML files, and the simulator supports a plugin system for extending functionality.

Data Flow

  1. Scenario Loading — The scenario loader parses a JSON/YAML file and instantiates Node, Link, and Interface objects.
  2. Simulation Loop — The Controller advances the Clock each tick. On every tick, the EventBus dispatches events to all subscribers.
  3. Protocol Processing — When a node needs to send a packet, the IP layer determines the route, ARP resolves the MAC address, and the packet traverses the link with configured latency/loss.
  4. Plugin Hooks — Plugins receive on_tick and on_packet callbacks for custom logic.
  5. Telemetry Streaming — Every simulation event is pushed via WebSocket to connected dashboard clients.
  6. Frontend Rendering — The dashboard updates the topology graph, statistics charts, and event log in real time.

Core Models

The core module defines the fundamental data structures used throughout the simulator.

Packet

The Packet model represents a network frame/packet traversing the simulated network.

from typing import Optional

Simulation Engine

Controller

The Controller is the central orchestrator. It manages the simulation lifecycle:

python 
class Controller:
    async def load_scenario(self, path: str) -> None:
        """Parse scenario file and build topology."""

    async def start(self) -> None:
        """Begin the simulation loop."""

    async def stop(self) -> None:
        """Pause the simulation."""

    async def step(self) -> None:
        """Advance exactly one tick."""

    async def reset(self) -> None:
        """Reset simulation to initial state."""

    async def tick(self) -> None:
        """Process one simulation tick:
        1. Advance clock
        2. Process queued packets on all links
        3. Run protocol handlers
        4. Notify plugins
        5. Emit telemetry events
        """

Clock

The simulation uses a discrete tick-based clock:

python 
class Clock:
    tick: int = 0                    # Current tick count
    tick_rate: float = 0.1           # Seconds per tick (real-time delay)
    running: bool = False            # Is simulation running?

    def advance(self) -> int:
        """Increment tick and return new value."""
        self.tick += 1
        return self.tick
  • tick_rate — Controls real-time speed. 0.1 = 10 ticks/second.
  • The clock can be paused, resumed, or single-stepped.

EventBus

The EventBus implements a publish/subscribe pattern for loose coupling:

python 
class EventBus:
    def subscribe(self, event_type: str, callback: Callable) -> None:
        """Register a callback for an event type."""

    def emit(self, event_type: str, data: dict) -> None:
        """Broadcast event to all subscribers."""

    def clear(self) -> None:
        """Remove all subscriptions."""

Protocol Stack

ARP (Address Resolution Protocol)

ARP resolves IP addresses to MAC addresses within a subnet.

Behavior

  1. When a node needs to send to an IP on the same subnet but doesn't have the MAC, it broadcasts an ARP_REQUEST (dst_mac: ff:ff:ff:ff:ff:ff).
  2. All nodes on the link segment receive the broadcast.
  3. The target node replies with an ARP_REPLY containing its MAC.
  4. The sender caches the mapping in its ARP table.
  5. ARP entries expire after a configurable timeout (default: 300 ticks).
python 
class ArpTable:
    entries: dict[str, ArpEntry]     # IP → (MAC, expiry_tick)

    def lookup(self, ip: str) -> Optional[str]:
        """Return MAC for IP, or None if not cached."""

    def update(self, ip: str, mac: str, ttl: int = 300) -> None:
        """Add or refresh an ARP entry."""

IP (Internet Protocol)

The IP layer handles addressing and forwarding decisions:

  1. Same subnet — If dst_ip is on the same subnet as the sending interface, resolve via ARP and send directly.
  2. Different subnet — Forward to the default gateway (hosts) or consult the routing table (routers).
  3. TTL check — Decrement TTL on each hop. Drop packet if TTL reaches 0.
  4. No route — If no matching route is found, drop the packet and emit packet_dropped.

ICMP (Internet Control Message Protocol)

ICMP provides diagnostic functionality (ping):

  • Echo Request — Generated by the ping action. Sends an ICMP_ECHO packet to the target IP.
  • Echo Reply — When a node receives an ICMP_ECHO addressed to itself, it responds with an ICMP_REPLY.
  • TTL Exceeded — When a router drops a packet due to TTL=0, it sends a TTL exceeded message back to the source.
python 
# Ping action generates these packets
echo_request = Packet(
    type=PacketType.ICMP_ECHO,
    src_ip="192.168.1.10",
    dst_ip="192.168.1.20",
    payload={"seq": 1, "timestamp": current_tick}

Routing Engine

Static Routing

Routers use static routes defined in the scenario file. Each route specifies:

python 
class Route(BaseModel):
    network: str          # CIDR notation, e.g., "10.0.2.0/24"
    next_hop: str         # Next-hop IP address
    interface: str        # Outgoing interface name
    metric: int = 0       # Route preference (lower = preferred)

Longest Prefix Match (LPM)

When forwarding a packet, the router consults its routing table:

  1. Convert dst_ip to binary
  2. Compare against all route network prefixes
  3. Select the route with the longest matching prefix
  4. If multiple routes have the same prefix length, prefer the lowest metric
  5. If no route matches, drop the packet
python 
class RoutingEngine:
    def lookup(self, dst_ip: str, routes: list[Route]) -> Optional[Route]:
        """Find the best matching route via longest-prefix match."""
        best = None
        best_prefix_len = -1

        for route in routes:
            network, prefix_len = parse_cidr(route.network)
            if matches(dst_ip, network, prefix_len):
                if prefix_len > best_prefix_len:
                    best = route
                    best_prefix_len = prefix_len
                elif prefix_len == best_prefix_len:
                    if route.metric < best.metric:
                        best = route
        return best

Forwarding Table (Hosts)

Hosts have a simpler forwarding decision:

  • If dst_ip is on the same subnet → send directly (ARP resolve)
  • Otherwise → forward to the default gateway

Plugin System

NetSim supports a plugin system for extending simulator behavior without modifying core code. Plugins can observe events, inject packets, modify behavior, and collect metrics.

Creating a Plugin

  1. Create a new .py file in simulator/plugins/custom/
  2. Subclass SimPlugin
  3. Implement lifecycle hooks
python 
# simulator/plugins/custom/traffic_monitor.py

from simulator.plugins.base import SimPlugin

class TrafficMonitorPlugin(SimPlugin):
    """Counts packets per node and logs throughput statistics."""

    name = "traffic_monitor"
    version = "1.0.0"

    def __init__(self):
        self.packet_counts: dict[str, int] = {}
        self.byte_counts: dict[str, int] = {}

    async def on_load(self, context):
        """Called when the plugin is loaded.

        Args:
            context: Provides access to the simulation controller,
                     event bus, and topology.
        """
        self.logger = context.get_logger(self.name)
        self.logger.info(f"{self.name} v{self.version} loaded")

    async def on_tick(self, tick: int, context):
        """Called on every simulation tick.

        Args:
            tick: Current simulation tick number.
            context: Simulation context with access to all nodes/links.
        """
        if tick % 100 == 0:  # Log every 100 ticks
            self.logger.info(f"Tick {tick}: {self.packet_counts}")

    async def on_packet(self, event: str, packet, node, context):
        """Called when a packet event occurs.

        Args:
            event: Event type (packet_sent, packet_received, etc.)
            packet: The Packet object.
            node: The Node involved.
            context: Simulation context.
        """
        node_id = node.id
        self.packet_counts[node_id] = self.packet_counts.get(node_id, 0) + 1
        self.byte_counts[node_id] = self.byte_counts.get(node_id, 0) + packet.size

    async def on_unload(self):
        """Called when the plugin is unloaded or simulation resets."""
        self.logger.info(f"Final counts: {self.packet_counts}")

Plugin Lifecycle

graph LR A["Discovery"] --> B["Load"] B --> C["on_load"] C --> D["Running"] D --> E["on_tick / on_packet"] E --> D D --> F["on_unload"] F --> G["Removed"]
  1. Discovery — The PluginManager scans simulator/plugins/custom/ for .py files
  2. Load — Classes subclassing SimPlugin are instantiated
  3. on_load — Plugin receives the simulation context
  4. Running — Plugin receives on_tick and on_packet callbacks
  5. on_unload — Plugin cleans up when simulation resets or shuts down

Plugin Base Class

python 
class SimPlugin:
    name: str = "unnamed_plugin"
    version: str = "0.0.0"

    async def on_load(self, context) -> None: ...
    async def on_tick(self, tick: int, context) -> None: ...
    async def on_packet(self, event: str, packet, node, context) -> None: ...
    async def on_unload(self) -> None: ...

CLI Reference

text 
Usage: python -m simulator [OPTIONS]

Options:
  --scenario PATH    Path to scenario file (JSON or YAML)
                     Default: None (starts with empty topology)

  --host HOST        WebSocket server bind address
                     Default: 0.0.0.0

  --port PORT        WebSocket server port
                     Default: 8765

  --headless         Run without WebSocket server (batch mode)
                     Useful for testing or scripted scenarios

  --ticks N          Run for exactly N ticks then exit
                     Default: unlimited (runs until stopped)

  --daemon           Run as a background daemon (24/7 mode)
                     Detaches from terminal, writes PID file

  --speed FLOAT      Initial speed multiplier
                     Default: 1.0

  --log-level LEVEL  Logging level (DEBUG, INFO, WARNING, ERROR)
                     Default: INFO

Examples

bash 
# Basic: run with a scenario
python -m simulator --scenario simulator/scenarios/simple_lan.json

# Custom port
python -m simulator --scenario simulator/scenarios/simple_lan.json --port 9000

# Headless batch run (100 ticks, no UI)
python -m simulator --scenario simulator/scenarios/multi_router.json \
    --headless --ticks 100

# Daemon mode (24/7 operation)
python -m simulator --scenario simulator/scenarios/simple_lan.json --daemon

# Fast simulation (5x speed)
python -m simulator --scenario simulator/scenarios/simple_lan.json --speed 5.0

# Debug logging
python -m simulator --scenario simulator/scenarios/simple_lan.json \
    --log-level DEBUG

# Using the installed entry point
netsim --scenario simulator/scenarios/simple_lan.json

Deployment Guide

Running Locally (24/7)

Using systemd (Recommended)

Create a service file at /etc/systemd/system/netsim.service:

ini 
[Unit]
Description=NetSim Network Simulator
After=network.target

[Service]
Type=simple
User=netsim
Group=netsim
WorkingDirectory=/opt/netsim
ExecStart=/opt/netsim/venv/bin/python -m simulator \
    --scenario simulator/scenarios/simple_lan.json \
    --port 8765
Restart=always
RestartSec=5
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
bash 
# Enable and start
sudo systemctl daemon-reload
sudo systemctl enable netsim
sudo systemctl start netsim

# Check status
sudo systemctl status netsim

# View logs
journalctl -u netsim -f

Using nohup

bash 
nohup python -m simulator --scenario simulator/scenarios/simple_lan.json \
    --port 8765 > netsim.log 2>&1 &

# Save the PID
echo $! > netsim.pid

# Stop it later
kill $(cat netsim.pid)

Log Rotation

Create /etc/logrotate.d/netsim:

text 
/opt/netsim/netsim.log {
    daily
    rotate 7
    compress
    delaycompress
    missingok
    notifempty
    create 0640 netsim netsim
    postrotate
        systemctl restart netsim
    endscript
}

Known Limitations

LimitationDetails
Static routing onlyNo dynamic routing protocols (OSPF, BGP, RIP)
No TCP/UDPTransport layer is not simulated; only ICMP
No DHCPIP addresses must be statically assigned in scenarios
Single-threadedUses Python asyncio (single event loop)
No VLANsSwitch does not support 802.1Q VLAN tagging
No QoSNo traffic prioritization or queuing disciplines
No IPv6Only IPv4 is supported
No persistent stateSimulation state is lost on restart
WebSocket onlyNo REST API for external integration

Roadmap

Short-term (v1.x)

  • DHCP Protocol — Dynamic IP address assignment
  • TCP/UDP Transport Layer — Connection-oriented and datagram protocols
  • Packet Capture Export — Save packet traces in PCAP format
  • REST API — HTTP API alongside WebSocket for integration
  • Scenario Editor — Visual drag-and-drop topology builder in the dashboard

Medium-term (v2.x)

  • Dynamic Routing — OSPF — Link-state routing with area support
  • Dynamic Routing — BGP — Border Gateway Protocol for AS peering
  • VLAN Support — 802.1Q tagging and trunk ports
  • STP — Spanning Tree Protocol for loop prevention
  • NAT — Network Address Translation on routers

Long-term (v3.x)

  • Multi-user Collaboration — Multiple users controlling the same simulation
  • Docker Deployment — Containerized simulator with docker-compose
  • IPv6 Support — Dual-stack networking
  • QoS & Traffic Shaping — Priority queues, rate limiting, DiffServ
  • SDN Controller Integration — OpenFlow-style programmable forwarding
  • AI-Powered Troubleshooting — LLM-assisted network diagnostics
